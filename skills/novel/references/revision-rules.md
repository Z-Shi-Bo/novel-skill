# Revision Rules

严格区分以下动作词：

| 模式 | 定义 | 默认边界 |
|---|---|---|
| polish | 润色 | 不改结构主干 |
| revise | 改写 | 允许局部重组 |
| rewrite | 重写 | 允许大改 |
| expand | 扩写 | 保持主干不变，补细节与场面 |

## 默认顺序

1. 先确认用户要的是哪一种。
2. 再明确修改范围：句子 / 段落 / 场景 / 全章。
3. 改稿前先读当前正式稿、章纲、状态卡和最近摘要。
4. 涉及结构级改动时，先出最小修法，再执行。

## 审查请求与改稿请求的区别

- 用户说“检查这章有没有问题” → 默认只出审查报告，不直接改正文。
- 用户说“顺手帮我改” → 可以在审查后继续执行修改。

## 自动修改边界

可自动做：
- 语病修整
- 重复句式和套话替换
- 标点与格式清理
- 小范围语言精炼

不默认自动做：
- 大改正式正文
- 改变角色核心动机
- 重写关键冲突
- 覆盖设定圣经
