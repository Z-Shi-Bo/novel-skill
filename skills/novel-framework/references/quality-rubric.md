# Quality Rubric

## 四层审查

### Layer 1：任务完成
- 是否完成章纲目标
- 是否推进主线 / 支线之一
- 是否有明确状态变化
- 是否有章末挂钩或阶段结算

### Layer 2：一致性
- 是否 OOC
- 是否吃设定
- 是否有时间线冲突
- 状态卡是否与正文吻合
- 关系、资源、伤势、情报是否连续

### Layer 3：阅读体验
- 节奏是否拖沓
- 信息密度是否失衡
- 冲突是否有效
- 对话是否空转
- 结尾是否具备继续阅读动力

### Layer 4：语言与风格
- 是否有 AI 味
- 是否有句式 / 词汇重复
- 是否口号化、套话化
- 是否缺少动作、物件、感官细节
- 是否偏离项目既定风格

## 问题分级

| 级别 | 含义 | 处理 |
|---|---|---|
| P0 | 阻断级 | 未修不能进入下一章 |
| P1 | 高优先级 | 原则上本章定稿前修复 |
| P2 | 优化级 | 可并入润色阶段 |

## P0 常见示例

- 重大设定冲突
- 明显 OOC
- 时间线断裂
- 章纲任务完全未完成
- 当前状态卡与正文严重脱节
- 关键逻辑崩坏

## 章节完成定义

一章只有在以下条件都满足时，才算完成：

- 正式稿存在
- review 完成
- 无 P0
- 摘要完成
- 状态卡同步
- 伏笔池同步
- 诊断通过
